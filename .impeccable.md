# MoltTrade Design Context

## Design Context

### Users
投机者 / 预测市场爱好者，不一定是 crypto 原生用户。他们打开 MoltTrade 是为了：
- 快速浏览热门事件的市场赔率
- 借助 AI 多视角分析做出更有把握的投注决策
- 管理自己的持仓和 P&L

使用场景：高专注度、短会话。用户往往在想下注前来验证自己的判断，或寻找 edge。

### Brand Personality
**Confident · Avant-garde · Sharp**

MoltTrade 是一个聪明、有态度的工具，不是一个讨好用户的仪表板。语气直接、克制、专业，带着一点边缘感——像一个精准的分析师，也像一本有观点的杂志。

### Emotional Goal
打开界面时应感受到：**边缘感 + 兴奋感**——类似进入交易所或高档赌场，不是紧张的恐惧，而是专注的肾上腺素。

### Aesthetic Direction
**暗色主调** + **单一克制的 accent color**（首选锐利的绿色，与 YES/profit 语义统一）。
整体是黑色打底的 "editorial trading terminal" 风格：
- 大量黑/zinc 负空间
- 高对比度文字层级
- 数字和数据用 tabular 字体
- 线条和边框取代色块来划分区域
- 动效克制但精准（状态变化用过渡，不用装饰动画）

**参考**: Polymarket（信息密度、市场卡片结构）、Bloomberg Terminal（数据层级），以及高端加密交易平台的冷静感。

**反参考**:
- ❌ DeFi 土狗 UI：霓虹渐变 + 宇宙星空背景
- ❌ AI 模板风格：玻璃卡片 + 紫到蓝渐变 + Sparkles 图标
- ❌ 传统金融（彭博/路透）的老气蓝灰
- ❌ 圆乎乎的 SaaS dashboard 风格

### Accent Color
**Sharp Green** — 与 YES/profit/上涨语义统一，在纯黑背景上对比强烈。
推荐值：`#16c784`（Polymarket 的 YES 绿，经过实战验证）或 `oklch(72% 0.22 150)`
- 只用于：YES 方向、正 P&L、主 CTA、关键 hover 状态
- 不普遍使用，保持稀缺/强调性

### Design Principles
1. **Data 优先，装饰靠后** — 用户来看数字。每个像素都要服务于信息传达，不用装饰填空间。
2. **边框胜过色块** — 用 1px 线条划分区域，保留大量黑色空间，视觉更干净、更专业。
3. **层级极简** — 最多 3 层视觉权重：主要内容 / 次要标记 / 背景注释。避免嵌套卡片。
4. **绿色稀缺** — accent 颜色只出现在真正重要的地方（YES/profit/CTA），稀缺性保证视觉冲击力。
5. **冷静的动效** — 入场动画控制在 400ms 以内，easing 用 ease-out，不用弹簧或弹跳效果。
